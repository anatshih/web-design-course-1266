الفكرة العامة

تطبيق ويب تفاعلي متكامل يهدف إلى إدارة المهام اليومية. يسمح للمستخدم بإضافة المهام وتعديلها وحذفها وتغيير حالتها مع واجهة جذابة وسهلة الاستخدام.

المتطلبات الأساسية (40%)

الوظائف الأساسية التي يجب أن يحتويها التطبيق:

إدارة المهام:
  • إضافة مهمة جديدة
  • تعديل مهمة موجودة
  • حذف مهمة
  • تغيير حالة المهمة (مكتملة أو غير مكتملة)
بيانات المهمة - يجب أن ينزخ النظام البيانات التالية:
  • العنوان: نص إجباري لا يجوز تركه فارغاً
  • مستوى الأولوية: قائمة منسدلة إجبارية (منخفضة / متوسطة / عالية)
  • التاريخ: حقل اختياري لتنظيم العمل
  • الحالة: مكتملة أو غير مكتملة
العرض والتخزين:
  • عرض جميع المهام بوضوح على الشاشة
  • تمييز المهام المكتملة (بخط مشطوب أو لون مختلف)
  • الحفظ التلقائي في LocalStorage
  • استرجاع المهام عند فتح الصفحة بدون إعادة تحميل

التقنيات المستخدمة

HTML5 CSS3 JavaScript LocalStorage DOM Manipulation Array Methods

المتطلبات المتوسطة (25%)

البحث: توفير شريط بحث للبحث عن مهمة من خلال كلمات من عنوانها
الفلترة: أزرار أو خيارات لتصفية عرض المهام:
  • حسب الحالة (مكتملة / غير مكتملة)
  • حسب مستوى الأولوية (عالية / متوسطة / منخفضة)
الترتيب: إمكانية ترتيب المهام المعروضة:
  • حسب التاريخ (الأحدث أو الأقدم)
  • حسب الأولوية (الأهم أولاً)

مميزات التميز - Bonus (10%)

يمكنك اختيار واحدة أو أكثر من المميزات التالية لإظهار مهاراتك المتقدمة:

إحصائيات: شريط تقدم يعرض نسبة المهام المكتملة (مثلاً: 3 من 5)
تحسينات بصرية:
  • رسوم متحركة عند الإضافة والحذف
  • تأثيرات بصرية احترافية
تفاعل متقدم: سحب وإفلات (Drag & Drop) لإعادة ترتيب المهام
تنبيهات وتنويهات:
  • رسائل تأكيد قبل الحذف
  • تنبيهات عند إضافة أو تعديل المهام

البنية البرمجية - كيفية كتابة الكود

لتنظيم الكود وتسهيل قراءته:

استخدام Array: استخدم "مصفوفة" (Array) لتخزين قائمة المهام بالكامل
استخدام Object: استخدم "كائن" (Object) لتمثيل بيانات كل مهمة على حدة
مثال توضيحي للبيانات: 
const task = {
  id: 1,
  title: "إنهاء المشروع",
  priority: "عالي",
  date: "2024-12-31",
  completed: false
};
نصيحة برمجية: قسم الكود إلى دوال (Functions) منفصلة لكل وظيفة، استخدم أسماء متغيرات واضحة وذات معنى، وأضف تعليقات توضح آلية عمل الكود

خطة العمل (المراحل التطبيقية)

اتبع هذه المراحل بهذا الترتيب لضمان إنجاز المشروع بسلاسة:

المرحلة المهام المطلوبة
1. التخطيط والتصميم رسم وتخيل شكل الواجهة، تحديد التطلبات والعناصر الأساسية، تصميم الواجهة
2. بناء الهيكل كتابة الكود باستخدام HTML، تنسيق العناصر وترتيبها باستخدام CSS
3. البرمجة الأساسية كتابة كود JavaScript لتفعيل وظائف الإضافة والحذف والتعديل والتغيير الحالة
4. الفلترة والبحث دمج وظائف البحث والفلترة والترتيب حسب الخصائص
5. الاختبار إجراء اختبار شامل للتطبيق على المتصفح والتأكد من عدم وجود أخطاء
6. التقرير والتسليم كتابة التقرير النهائي وتجهيز الملفات للتسليم

الفكرة العامة

تطبيق ويب تفاعلي يعمل من خلال المتصفح يشبه تطبيقات كتابة الملاحظات في الهواتف الذكية. يسمح للمستخدم بإضافة الملاحظات وتعديلها وحذفها مع تصنيفها في فئات مختلفة والبحث فيها بسهولة. يتميز هذا المشروع بأنه يركز على إدارة النصوص والبحث السريع.

المتطلبات الأساسية (40%)

عمليات CRUD (الأساسية):
  • Create: إضافة ملاحظة جديدة
  • Read: عرض جميع الملاحظات بوضوح
  • Update: تعديل ملاحظة موجودة
  • Delete: حذف ملاحظة
بيانات الملاحظة - يجب أن ينزخ النظام:
  • العنوان: نص إجباري لا يجوز تركه فارغاً
  • المحتوى: نص الملاحظة - يجب أن يستوعب نصوص طويلة
  • الفئة: تصنيف الملاحظة (دراسة، عمل، شخصي)
  • تاريخ الإنشاء: يتم إنشاؤه وحفظه تلقائياً بدون تدخل المستخدم
  • تاريخ التعديل: يتحدث تلقائياً عند تعديل الملاحظة
طريقة العرض والتخزين:
  • عرض الملاحظات في شكل بطاقات (Cards)
  • تحديث الواجهة بشكل مباشر (بدون إعادة تحميل الصفحة)
  • حفظ البيانات في LocalStorage
  • استرجاع الملاحظات عند فتح الصفحة

التقنيات المستخدمة

HTML5 CSS3 JavaScript Real-time Search LocalStorage DOM Manipulation

المتطلبات المتوسطة (20%)

البحث الفوري (Real-time Search):
  • يجب أن يحتوي التطبيق على عبرم بحث
  • يبحث في "العنوان" و "المحتوى"
  • البحث يعمل فوراً أثناء كتابة المستخدم (حدث input)
  • ليس بعد الضغط على زر بحث
التصنيف والفلترة:
  • تصنيف الملاحظات عند إضافتها في الفئات (دراسة، عمل، شخصي)
  • أزرار أو خيارات لتصفية العرض
  • مثلاً: إظهار ملاحظات "الدراسة" فقط، أو إظهار "الجميع"
الترتيب: إمكانية ترتيب الملاحظات المعروضة:
  • حسب التاريخ (الأحدث/الأقدم)
  • حسب الاسم (أبجدياً)

مميزات التميز - Bonus (10%)

يمكنك اختيار واحدة أو أكثر من المميزات التالية:

للإدارة:
  • زر تثبيت الملاحظة (Pin) في أعلى القائمة
  • تمييزها بنجمة (مفضلة)
  • تغيير لون البطاقة
للإحصائيات:
  • عرض عدد الملاحظات الكلي
  • عدد الملاحظات لكل فئة
  • إحصاء لعدد الكلمات
للواجهة:
  • وضع ليلي/نهاري (Dark Mode)
  • تصميم متجاوب يعمل على الهواتف
  • رسوم متحركة عند الحذف والإضافة
  • تنبيهات احترافية

البنية البرمجية - كيفية كتابة الكود

هيكل البيانات:
  • نزخ الملاحظات في Array
  • كل ملاحظة هي Object منفصل
مثال توضيحي للبيانات: 
const note = {
  id: 1,
  title: "ةمهم ةظحلام",
  content: "...انه ةظحلاملا ىوتحم",
  category: "ةسارد",
  createdDate: "2024-12-15",
  lastModified: "2024-12-15 10:30"
};
نصيحة برمجية: قسم الكود إلى دوال صغيرة (Functions) - دالة لإضافة، دالة للبحث، دالة للعرض، إلخ. هذا يجعل الكود أسهل للقراءة والتعديل

خطة العمل (المراحل التطبيقية)

المرحلة المهام المطلوبة
1. التخطيط والتصميم تخيل شكل الصفحة (أين ثحب البحث؟ أين الملاحظات؟) وتحديد الألوان والتصميم

3. البرمجة الأساسية تفعيل وظائف الإضافة والعرض والحذف والتعديل (CRUD)

5. البحث والفلترة تفعيل شريط البحث الفوري وخيارات الفلترة والترتيب
