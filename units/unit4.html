🧩 أولاً: واجهة المستخدم الرسومية GUI

GUI هي العناصر التي يراها المستخدم ويتفاعل معها، مثل الأزرار وحقول النصوص والقوائم ومربعات الاختيار.

Button Text Input Textarea Checkbox Select

ما المقصود بـ GUI؟

هي الواجهة التي تسمح للمستخدم بإدخال البيانات أو اختيار أمر أو مشاهدة نتيجة. كل عنصر يمكن التعامل معه برمجيًا.

لماذا نستخدمها؟

  • لتسهيل التفاعل مع الصفحة.
  • لإدخال البيانات والحصول على نتائج.
  • لربط العناصر بأحداث مثل النقر أو التغيير.

أهم فكرة يجب فهمها

العناصر ليست مجرد شكل، بل هي كائنات لها خصائص:

  • value — قيمة حقل النص
  • checked — حالة مربع الاختيار
  • selectedIndex — الخيار المحدد من القائمة

وعند تفاعل المستخدم، تستطيع JavaScript قراءة هذه القيم.

ملخص

GUI هو الجزء المرئي الذي يتعامل معه المستخدم. عناصر الواجهة ليست ثابتة، بل يمكن قراءتها والتحكم بها عبر JavaScript.

التالي: التحقق من البيانات ⬅

✅ ثانياً: التحقق من صحة البيانات Form Validation

قبل إرسال البيانات، يجب التأكد أنها موجودة وصحيحة ومنطقية. مثلًا لا يجوز أن يكون العمر فارغًا أو نصًا أو عددًا غير منطقي.

length isNaN() حقول إجبارية مدى القيم

لماذا نتحقق من البيانات؟

  • لمنع إرسال بيانات ناقصة أو خاطئة.
  • لتقليل العبء على الخادم.
  • لإخبار المستخدم بالخطأ فورًا.

ما الثلاثة فحوصات الأساسية؟

  • 1️⃣ هل الحقل فارغ؟
  • 2️⃣ هل القيمة من النوع الصحيح (رقم أو نص)؟
  • 3️⃣ هل القيمة ضمن المدى المنطقي؟

✅ صحيح — فحص الفراغ أولاً
if (age === "") { ... } else if (isNaN(age)) { ... } // الآن نفحص الفراغ قبل النوع

⚡ ثالثاً: صفحات الويب الديناميكية DHTML

DHTML ليس لغة مستقلة، بل مصطلح يشير إلى استخدام HTML + CSS + JavaScript + DOM معًا لجعل الصفحة تتغير دون إعادة تحميلها.

HTML CSS JavaScript DOM

الفكرة الأساسية

الصفحة الديناميكية لا تبقى ثابتة، بل تتغير عند التفاعل: قد يتغير نص أو لون أو حجم أو موضع، دون الحاجة لفتح صفحة جديدة.

ما مكونات DHTML؟

  • HTML — يبني هيكل الصفحة
  • CSS — ينسّق الشكل والألوان
  • JavaScript — يضيف التفاعل والمنطق
  • DOM — يسمح بتعديل العناصر لحظيًا

بعض الطلبة يخلطون بين تعديل جزء من الصفحة وبين استبدال الصفحة كاملة. DHTML يركز على تعديل جزء محدد داخل الصفحة الحالية، ليس فتح صفحة جديدة.

⚡ راجع بسرعة
  • DHTML ليس لغة مستقلة.
  • يجعل الصفحة أكثر حيوية دون إعادة تحميل.
  • يعتمد على HTML + CSS + JS + DOM معًا.
ملخص

DHTML هو نتيجة عمل HTML وCSS وJavaScript وDOM معًا. النتيجة: صفحة تتغير وتتفاعل دون الحاجة لإعادة التحميل.

➡ السابق: Validation التالي: DOM ⬅

🧠 رابعاً: نموذج كائن الوثيقة DOM

DOM يسمح لـ JavaScript بالتعامل مع عناصر الصفحة على أنها كائنات لها خصائص وإجراءات. الأداة الأساسية هي: document.getElementById()

getElementById() innerHTML src width / height style

ما الذي يقدمه DOM؟

  • الوصول لأي عنصر في الصفحة بالـ id.
  • تغيير النصوص والصور والأحجام والألوان.
  • صفحات ديناميكية دون إعادة تحميل.

الشكل العام لاستخدام DOM

// خطوة 1: الوصول للعنصر بالـ id let el = document.getElementById("myId"); // خطوة 2: تعديل خاصية el.innerHTML = "نص جديد"; el.style.color = "red"; el.src = "newImage.jpg";

كتابة اسم id بشكل مختلف بين HTML وJavaScript، مثل Title في مكان وtitle في مكان آخر. الـ id حساس لحالة الأحرف.

❌ خطأ — اختلاف الأحرف

⚡ راجع بسرعة

  • نصل للعنصر عبر getElementById(id).
  • نغيّر النص أو الحجم أو اللون أو الصورة.
ملخص

DOM هو الجسر الذي يربط JavaScript بعناصر الصفحة. من خلاله نستطيع الوصول لأي عنصر وتغيير خصائصه.

🖱️ خامساً: التعامل مع الأحداث Events

الحدث هو شيء يحدث داخل الصفحة، مثل تحميل الصفحة، أو النقر، أو مرور الفأرة. عند وقوع الحدث، تُستدعى دالة برمجية.

onload onclick onmouseover onmouseout onmousedown onmouseup

أحداث الصفحة

  • onload — عند انتهاء تحميل الصفحة
  • onunload — عند مغادرة الصفحة

تُستخدم لتشغيل رسالة ترحيب أو تهيئة عناصر عند فتح الصفحة.

أحداث الفأرة

  • onclick — عند النقر
  • onmouseover — عند مرور المؤشر فوق العنصر
  • onmouseout — عند ابتعاد المؤشر
  • onmousedown — عند الضغط المستمر
  • onmouseup — عند رفع الضغط

onmousedownعند الضغط والاستمرار<button onmousedown="hold()">
onmouseupعند رفع الإصبع<button onmouseup="release()">

➡ السابق: DOM التالي: Animation ⬅

🎈 سادساً: الكائنات المتحركة Animation

الحركة في صفحات الويب تعني تغيير موقع أو حجم عنصر مرات متتابعة بسرعة مناسبة. نستخدم setTimeout لتكرار الاستدعاء وclearTimeout للإيقاف.

setTimeout() clearTimeout() style.left / style.right style.top width / height

كيف تحدث الحركة؟

الحركة لا تعني أن العنصر يتحرك وحده، بل نغيّر موضعه أو حجمه مرات متتابعة بسرعة، فيبدو للمستخدم وكأنه يتحرك.

مثال: إذا غيّرنا style.right من 10 إلى 15 إلى 20... كل 30 ثانية، يبدو العنصر متحركًا.

الأداتان الأساسيتان

  • setTimeout(دالة, وقت) — ينفّذ الدالة بعد مدة (بالميلي ثانية)
  • clearTimeout(timer) — يوقف التوقيت ويوقف الحركة
// setTimeout يستدعي الدالة مرة بعد مرة timer = setTimeout(move, 30); // كل 30ms // clearTimeout يوقف الحركة clearTimeout(timer);

let timer = null; // مؤشر التوقيت let pos = 14; // الموقع الابتدائي (من اليمين) let dir = 1; // الاتجاه: 1 يسار، -1 يمين function move() { pos += 6 * dir; // نحرك بـ 6 بكسل if (pos >= 300) dir = -1; // وصل لليسار → انعكس if (pos <= 14) dir = 1; // وصل لليمين → انعكس // نطبّق الموقع الجديد على العنصر document.getElementById("ball").style.right = pos + "px"; // نستدعي نفسنا بعد 35ms → حركة مستمرة timer = setTimeout(move, 35); } function startMove() { if (!timer) move(); } function stopMove() { clearTimeout(timer); timer = null; }
📋 كود كامل قابل للتشغيل
<!DOCTYPE html> <html lang="ar" dir="rtl"> <head> <meta charset="UTF-8"> <title>Animation Example</title> </head> <body> <!-- مساحة الحركة --> <div style="position:relative;height:150px;border:2px solid #ccc;border-radius:8px;overflow:hidden;"> <div id="ball" style="position:absolute;right:10px;top:45px; width:50px;height:50px;border-radius:50%;background:orange;"> </div> </div> <button onclick="startMove()">▶ ابدأ</button> <button onclick="stopMove()">⏹ أوقف</button> <script> let timer = null; let pos = 10; let dir = 1; function move() { pos += 5 * dir; if (pos > 300) dir = -1; if (pos < 10) dir = 1; document.getElementById("ball").style.right = pos + "px"; timer = setTimeout(move, 30); } function startMove() { if (!timer) move(); } function stopMove() { clearTimeout(timer); timer = null; } </script> </body> </html>
⚠️ خطأ شائع

إذا لم توقف التوقيت باستخدام clearTimeout وضغطت على "ابدأ" مرتين، تتضاعف الحركة وتصبح سريعة جدًا لأن هناك دالتين تعملان في نفس الوقت!

❌ خطأ — لا فحص قبل البدء
function startMove() { move(); // إذا ضغطت مرتين تضاعفت! }
✅ صحيح — فحص أن لا يوجد timer نشط
function startMove() { if (!timer) move(); // تأكد أنها لم تبدأ }
⚡ راجع بسرعة
  • الحركة = تغيير متكرر للموقع أو الحجم.
  • setTimeout يستدعي الدالة مرة بعد مرة.
  • clearTimeout يوقف الحركة.
  • افحص دائمًا أن الـ timer غير نشط قبل البدء.
ملخص

الحركة في صفحات الويب تعتمد على تغيير متكرر للموقع أو الحجم مع التحكم بالزمن عبر setTimeout وclearTimeout.

📝 المراجعة النهائية للوحدة

إذا فهمت النقاط التالية فأنت على الطريق الصحيح:

  • تعرفت على عناصر الواجهة GUI وكيف يتفاعل المستخدم معها.
  • فهمت كيف نتحقق من صحة البيانات بثلاث مراحل.
  • أدركت أن DHTML هو استخدام HTML+CSS+JS+DOM معًا.
  • تعلمت أن DOM يمكّننا من الوصول للعناصر وتغييرها.
  • فهمت دور الأحداث مثل onclick وonmouseover وonload.
  • رأيت كيف تحدث الحركة عبر setTimeout وclearTimeout.
🧩 اختبار قصير نهائي

1) أي مفهوم يتيح لنا تغيير نص عنصر داخل الصفحة؟

2) أي حدث يناسب تنفيذ أمر عند النقر على زر؟

3) أي دالة مناسبة لفحص أن القيمة ليست رقمًا؟

4) أي دالة تستخدم لإيقاف الحركة؟

أنشئ صفحة تحتوي على حقل نص وزر، وعند الضغط يظهر النص داخل فقرة أسفل الزر.

تحقق أن المستخدم أدخل رقمًا فقط في حقل العمر.

💡 تلميح
// isNaN(value) ترجع true إذا كانت القيمة ليست رقمًا if (isNaN(value)) { ... }
<!DOCTYPE html> <html lang="ar" dir="rtl"> <body> <input id="age" placeholder="أدخل العمر"> <button onclick="check()">تحقق</button> <p id="result"></p> <script> function check() { let v = document.getElementById("age").value.trim(); if (v === "") { document.getElementById("result").innerHTML = "⚠️ الحقل فارغ"; } else if (isNaN(v)) { document.getElementById("result").innerHTML = "❌ أدخل رقمًا فقط"; } else { document.getElementById("result").innerHTML = "✅ العمر صحيح: " + v; } } </script> </body> </html>
